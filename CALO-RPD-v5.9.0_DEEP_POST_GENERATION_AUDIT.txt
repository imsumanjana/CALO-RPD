1. EXECUTIVE VERDICT
=====================================================================================

v5.9.0 is materially stronger than v5.8.0 in the exact areas identified by the v5.8 deep scientific audit.
The release now has a substantially more coherent scientific chain from policy training through CALO execution,
ORPD evaluation, robust/security scenarios, backend parity, formal qualification and publication evidence.

Most importantly, v5.9 does NOT merely rename the old policy ABI or add superficial guards. It changes the native
policy/controller semantics and therefore introduces an explicit new v5.9 semantic ABI. Training Champion evidence
is separated from deployable Base evidence, stale recovery is authority-checked, repair accounting is unified,
N-1 includes the intact system, branch angle limits are enforced, exact experiment PF/tolerance configuration is
included in accelerator parity, and formal non-inferiority is statistically defined rather than inferred from the
absence of superiority.

The post-generation audit itself found additional defects after the first v5.9 implementation pass. Those defects
were corrected before release rather than being hidden in a “completed” package. The most important self-audit
corrections were:

  1. hidden PPO-training versus deployed-CALO clock/RNG/memory/intervention mismatches;
  2. device-resident branch-angle endpoint indexing;
  3. scientifically incomplete cross-run accelerator batch signatures;
  4. a default PolicyQualifier.run() non-inferiority margin bug caused by referencing optional `config`
     instead of resolved validated `qconfig`;
  5. aggregate-only formal statistical gating, strengthened to comparator-by-case gating;
  6. dead PQ-bus generator-voltage decision variables;
  7. scenario transform structural-identity validation gaps;
  8. Experiment workspace prerequisite gating inconsistency;
  9. incomplete v5.9 freeze coverage for newly introduced scientific modules.

After those corrections, no known unresolved P0/P1 correctness defect from the v5.8 scientific-audit scope was
identified in the source paths that could be exercised in this build runtime.

That statement is intentionally narrower than “the software is proven bug-free.” Full PyQt6, PYPOWER and physical
CUDA/XPU validation are still unavailable in this runtime and remain mandatory target-environment gates.

3. v5.8 CRITICAL/HIGH FINDINGS — FINAL v5.9 STATUS
=====================================================================================

CT-001 — STALE RECOVERY AUTHORITY
STATUS: RESOLVED

Recovery metadata persists the parent manifest SHA-256 and parent generation ID. Before an interrupted session may
publish recovered branch state as authoritative, v5.9 recomputes/loads current authority and requires an exact
match. If another session has advanced authority, stale recovery fails closed with explicit fork/export guidance.

This prevents the v5.8 failure mode:
  G100 interrupted recovery -> later G200 success -> old G100 recovery silently replacing G200.

Dedicated stale-recovery regression: PASS.

-------------------------------------------------------------------------------------

POLICY-001 — TRAINING ENVIRONMENT != DEPLOYED CALO
STATUS: RESOLVED FOR THE NATIVE v5.9 SEMANTIC CONTRACT, WITH DECLARED TEST SCOPE

A new v5.9 policy ABI explicitly defines:
- one raw global neural regime/operator/parameter action;
- explicit environment/controller interventions;
- per-learner executed decision provenance.

During self-audit, hidden differences were found in archive/success-memory capacities, epsilon/progress clocks,
stagnation context, deterministic-policy handling of environmental randomness, probability handling, RNG
consumption, personal-best propagation and feasibility credit. These were corrected.

The final seeded multi-transition regression compares PPO environment transition state to actual deployed
`CALOOptimizer` and verifies equality of:
- population,
- personal best,
- optimizer RNG state,
- epsilon,
- operator contextual credit,
- memory contextual credit,
- reward,
- executed operators.

This is strong evidence for the tested transition kernel. It is not overclaimed as exhaustive proof over every
possible long-horizon/hardware/configuration trajectory.

-------------------------------------------------------------------------------------

POLICY-002 — FORCED RECOVERY CREDIT MISATTRIBUTION
STATUS: RESOLVED

Raw neural operator and final executed operator/intervention are separate. When forced recovery replaces ordinary
candidate generation, success is attributed to the executed recovery intervention rather than the overridden
neural operator.

POLICY-004/005/006 — CHAMPION/BASE EVIDENCE SCIENCE
STATUS: RESOLVED/MATERIALLY REDESIGNED

v5.9 separates:

A. Training Champion / provisional candidate
   Synthetic curriculum screening is allowed.

B. Deployable Scientific Base
   Requires exact real-ORPD development evidence under an explicit ExperimentConfig.

Deployable evidence includes exact:
- objective definition/scales,
- variable/control profile,
- robust formulation/scenarios,

- constraint tolerances.

Metrics distinguish:

Base eligibility uses final-best feasibility for every required real development case.

Raw heterogeneous task objectives are not used as one unqualified deployment median. Real-case objective evidence
is normalized by a fixed formulation-specific reference scale and retained per case.

STAT-001 — NON-INFERIORITY NOT A FORMAL TEST
STATUS: RESOLVED

NI margin; it now correctly uses the validated resolved `qconfig`.

- sparse Jacobian,
- transformer taps and phase shifts,
- branch complex flows / loading,
- Q-limit switching,
- individual P/Q capability accounting,
- mixed ORPD controls,
- fixed voltage-deviation/L-index partitions,
- active-loss / voltage-deviation / L-index / multiobjective modes,
- robust/scenario aggregation,
- N-1 base + selected contingencies,
- branch angle limits,
- accelerator parity,
- independent AC-PF cross-validation,
- reproducible experiment/seed/evidence workflow.

Declared scientific modeling conventions/limitations:

1. Co-located generator Q allocation
   Bus-level solved reactive requirement is distributed capability-proportionally for deterministic reporting and
   per-unit limit accounting. It is not a uniquely solved physical AVR participation model.

2. Load uncertainty
   Current model is aggregate/system-wide scaling, not a spatially correlated nodal stochastic forecast model.

3. Renewable uncertainty
   Current model is simplified net-load uncertainty, not a full inverter P-Q capability/curtailment/voltage-control
   stochastic model.

4. Independent validation
   PYPOWER is used as independent AC power-flow numerical cross-validation. It is not a second independent
   implementation of the complete ORPD decoder/repair/optimizer/robust formulation.

These are now documented instead of being overclaimed.

12. FINAL CONCLUSION
=====================================================================================

v5.9.0 is a genuine scientific-closure improvement over v5.8.0.

POLICY:
The native v5.9 policy system is now much more scientifically coherent because raw neural action, environmental
interventions, executed action, reward and provenance are explicitly separated and tested for seeded transition
parity. Deployable Base promotion no longer rests on synthetic-only evidence.

CALO:
CALO remains a scientifically rich custom optimizer. Fairness instrumentation is stronger because repair accounting
is path-independent and FE-budget rules are explicit. Publication superiority still requires empirical ablation and
paired benchmark evidence.

POWER SYSTEM / RPD:
The workflow is substantially more complete and accurate: N-1 includes the intact system, active branch angle limits
are enforced, dead PQ voltage controls are removed, PF/tolerances are part of parity, and sparse power-flow support is
stronger. Simplified uncertainty and co-located Q allocation are correctly documented as modeling conventions rather
than hidden “exact physics.”

POST-GENERATION AUDIT RESULT:
No known unresolved critical/high source-code defect from the v5.8 scientific-audit closure scope remains in the
paths that could be exercised in this runtime after the final self-audit corrections above.

RELEASE RECOMMENDATION:
v5.9.0 can be used as the corrected next research-validation baseline and as the source for target-machine release
qualification. It should be called a final publication freeze only after the missing PyQt6/PYPOWER/physical
accelerator/long-soak and frozen repeated-campaign gates are successfully completed.
