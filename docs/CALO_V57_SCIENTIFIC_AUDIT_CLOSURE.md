# CALO-RPD v5.7 — carried-forward v5.4 audit closure

v5.7 is a focused scientific/reproducibility hardening release. It targets the findings that the v5.6 re-audit still listed under “Previous v5.4 findings that still need resolution.” It does not claim that every new v5.6 competitive-training issue is solved; those later findings remain a separate remediation track.

## Closed scientific definitions

- Robust feasibility defaults to **all-scenario maximum constraint violation**. Expected-weighted violation remains available only as an explicitly selected formulation.
- Scenario weights are validated at the low-level scientific boundary; Scenario objects are immutable; exact-resume fingerprints include callable code identity, closure cells, defaults and keyword defaults.
- Generator active/reactive limits are accounted per individual online unit. Dynamic PV→PQ switching remains a numerical feasibility mechanism and does not redefine the fixed voltage-deviation or L-index mathematical bus partition.
- CPU Newton-Raphson uses vectorized Jacobian construction, sparse solve where available, deterministic dense fallback, and monotone backtracking.
- Boundary clipping is an explicit shared repair policy with per-run repair counts/rates.
- Strict equal-FE experiments require the requested FE budget to be divisible by the common population size; CALO and baselines therefore receive the same complete requested budget under the frozen comparison protocol.

## Policy qualification

Formal promotion defaults to 30 paired runs per development case, uses case-wise feasible-objective summaries, feasibility-first convergence AUC including pre-feasible delay, mandatory independent PF cross-validation, paired directional/effect evidence and Holm correction. Periodic in-training formal qualification is retired by design; formal qualification applies to saved Base artifacts under a separately budgeted campaign.

## Publication integrity

When independent validation is required, zero verified rows blocks export. Publication-facing tables/plots/statistics are built from the verified evidence set only. Best-solution artifacts require a feasible finite solution. Scenario feasibility requires both PF convergence and constraint feasibility. Publication readiness requires complete expected paired evidence across the compared algorithms.

## Restoration and continuation

Workspace restoration resolves an explicit scientific revision rather than update time, restores the exact saved PowerFlowOptions, blocks on failed PF restoration, preserves policy-binding diagnostics, and uses named panel identities rather than page indexes. Exact CALO continuation and paired recomputation remain explicitly distinct trajectories throughout revision/segment provenance.

## Trust and diagnostics

Policy suppression is project/database scoped. Portable policy import is separated from trusted-local exact-resume pickle state; exact resume requires local HMAC-authenticated integrity. Broad `except Exception: pass` suppression is prohibited by regression test. Evaluator calibration is labeled evaluator-only, while CALO records full end-to-end requested-FE throughput separately.

## Deliberately not relabeled as solved

The full CALO cognitive/control plane is still not wholly CUDA/XPU resident, and some irregular host-side cognition/memory loops remain. Full GUI/PYPOWER/physical accelerator validation depends on the target environment. Large orchestration modules remain structural maintainability debt even though scientific service boundaries are now explicit.
